LimitSoul(ProgressionManager, LazyStateBuilder, Int32, Boolean)

Changes the maximum soul cap, similar to the effect after death.
For example, if limiter is 33, the soul cap is reduced to 99 - 33 = 66. With consecutive uses, the new value is kept.
If the cap is increased, the new portion of the meter is treated as empty, and reserves are rebalanced.
If the soul cap appliesToPriorPath, this can fail if previous soul usage required a higher soul cap. If the lowered cap only applies going forward, this should not fail.

TrySpendSoul(ProgressionManager, ref LazyStateBuilder, Int32)

Determines whether SpendSoul(ProgressionManager, LazyStateBuilder, Int32) would succeed, and if so modifies the state and returns true.
On false return, the state is not modified.
Per its signature, the result is a single state. Implementations where SpendSoul(ProgressionManager, LazyStateBuilder, Int32) would produce multiple states must admit a consistent truncated result.
